# shadcn/ui 全量组件重构设计

日期：2026-06-18

## 1. 背景与目标

现有项目是 React 19、Vite 6、Tailwind CSS 4 的桌面端患者画像演示系统，包含工作台、标签中心、人群运营、预警管理、触达运营、数据服务、主题分析七个业务模块。

当前页面大量直接使用原生按钮、输入框、下拉框、表格和自定义弹层，组件样式与交互规则分散。部分页面文件较大，但本次不同时进行页面业务结构重写。

本次目标：

- 使用官方 shadcn/ui CLI 将所有交互控件迁移到项目内源码组件。
- 保留现有业务数据、文案、页面布局、信息密度和有效交互。
- 保留青色视觉主题，采用 New York 紧凑风格。
- 引入 HashRouter，替代全局自定义事件导航。
- 统一弹层、危险操作、表单反馈、焦点和键盘交互。
- 为核心组件和关键业务流程补充自动化测试。

## 2. 非目标

本次不包含：

- 移动端或平板适配。
- 深色模式。
- 真实后端 API 接入。
- TanStack Table。
- React Hook Form 或 Zod。
- Recharts 图表实现重写。
- 全面拆分大页面或进行无关架构重构。
- 扩大现有 `localStorage` 持久化范围。
- 补齐现有无行为占位按钮的业务功能。
- 完整 WCAG 2.2 AA 审计。

## 3. 技术方案

### 3.1 shadcn/ui 基础设施

- 使用官方 shadcn/ui CLI。
- 风格选择 New York。
- 底层交互采用 Radix UI。
- 使用 CSS Variables 管理基础主题。
- 仅安装实际使用的组件，不引入全量组件。
- 组件源码生成到 `src/components/ui/`，随项目提交并允许集中定制。
- 将 `@/*` 统一指向 `src/*`，同步调整 Vite 和 TypeScript alias，供 CLI 生成代码使用。
- 新增 `components.json`，记录风格、别名和 Tailwind CSS 配置。

预计使用的基础组件：

- `Button`
- `Input`
- `Textarea`
- `Select`
- `Dialog`
- `Sheet`
- `AlertDialog`
- `Checkbox`
- `Switch`
- `RadioGroup`
- `Tabs`
- `Table`
- `Badge`
- `Tooltip`
- `Sidebar`
- `Pagination`
- `Sonner`

如迁移过程中确认某个组件没有实际调用，不保留其生成源码和依赖。

### 3.2 业务通用组件

在 `src/components/common/` 增加少量稳定的业务通用组件：

- `PageHeader`：页面标题、副标题和右侧操作区。
- `StatusBadge`：统一业务状态到 Badge 变体和业务色的映射。
- `CompactPagination`：总数、页码、前后翻页的紧凑型分页。
- `DataTableShell`：表格标题、工具栏、滚动区域、空状态和底部分页容器。
- `ConfirmAction`：对 AlertDialog 的轻量业务封装。

只提取跨页面重复且接口稳定的结构。页面内部业务区块继续保留在原页面文件中，避免将组件迁移和大规模架构调整叠加。

### 3.3 基础组件定制边界

- 集中调整尺寸、圆角、边框、阴影和主题变量。
- 默认使用适配紧凑后台的小尺寸变体。
- 不在业务页面中复制修改基础组件内部实现。
- 不改变 Radix 提供的焦点管理、键盘操作和弹层行为。

## 4. 组件迁移规则

| 现有实现 | 目标实现 |
| --- | --- |
| 原生 `button` | shadcn `Button` |
| 原生文本输入 | shadcn `Input` |
| 原生多行输入 | shadcn `Textarea` |
| 原生 `select` | shadcn `Select` |
| 居中表单弹层 | shadcn `Dialog` |
| 右侧详情面板 | shadcn `Sheet` |
| 已有功能的删除、停用等危险确认 | shadcn `AlertDialog` |
| 多选字段 | shadcn `Checkbox` |
| 启停状态 | shadcn `Switch` |
| 渠道单选 | shadcn `RadioGroup` |
| 页面标签切换 | shadcn `Tabs` |
| 原生表格结构 | shadcn `Table` |
| 状态标签 | shadcn `Badge` 或 `StatusBadge` |
| 纯图标操作 | `Button` + `Tooltip` + `aria-label` |
| 页面侧栏 | shadcn `Sidebar` |
| 操作结果提示 | shadcn `Sonner` |

补充规则：

- 表格不引入 TanStack Table，继续使用现有页面状态处理分页、筛选和排序。
- Recharts 保留现有图表、Tooltip、Legend 和数据结构，仅统一颜色变量与容器样式。
- 现有无行为占位按钮迁移为 shadcn `Button`，但不增加反馈或数据变化。
- 危险操作统一确认仅适用于当前已经具备实际行为的操作，不为占位操作新增行为。
- 表单继续使用现有 `useState` 数据流。
- 校验错误显示在对应字段下方，并使用 Toast 汇总提交失败。
- 成功、失败和警告结果统一使用 Sonner Toast。

## 5. 视觉规范

### 5.1 设计基线

- 保留现有青色品牌主题和 slate 中性色。
- 使用 New York 风格的清晰边框、克制圆角和轻阴影。
- 保持现有桌面后台信息密度，避免采用偏宽松的默认控件尺寸。
- 仅提供浅色主题。

### 5.2 颜色

- 基础交互组件使用 `background`、`foreground`、`primary`、`muted`、`border`、`ring`、`destructive` 等语义变量。
- 风险等级、运行状态、预警级别和图表系列继续使用明确业务色。
- 逐步清理仅用于表达基础主题的硬编码色类，不强制将所有业务色改为语义变量。

### 5.3 动画

- 保留侧栏展开、Dialog、Sheet、Tooltip 和状态变化所需过渡。
- 移除不承担状态反馈的纯装饰动画。

## 6. 路由与数据流

### 6.1 路由表

应用使用 `HashRouter`：

| 模块 | 路径 |
| --- | --- |
| 工作台 | `/#/workbench` |
| 标签中心 | `/#/tags` |
| 人群运营 | `/#/population` |
| 预警管理 | `/#/warning` |
| 触达运营 | `/#/touchpoint` |
| 数据服务 | `/#/dataservice` |
| 主题分析 | `/#/analysis` |

根路径和未知路径重定向到工作台。

### 6.2 导航实现

- 侧栏使用 `NavLink` 生成菜单并判断激活状态。
- 保留现有侧栏展开和收起能力。
- 折叠状态只保存在当前 React 会话，刷新后恢复展开。
- 移除 `App.tsx` 中的 `activeMenu`、`menuParams` 和 switch 渲染。
- 移除全局 `appNavigate` 自定义事件及相关监听。

### 6.3 路由参数

- 可恢复的视图状态写入查询参数，例如搜索词和目标标签页。
- 跨模块创建触达任务、数据服务所需的临时对象通过 `navigate(path, { state })` 传递。
- 目标页面消费 `location.state` 后使用 replace 清除，避免刷新或返回时重复执行创建动作。
- 二级页面、Dialog 和 Sheet 继续使用模块内部状态，不建立子路由。

## 7. 数据与业务兼容

- 保留现有 mock 数据、业务文案和计算逻辑。
- 保留现有有效交互和状态流转。
- 保留现有持久化行为：已使用 `localStorage` 的快照数据继续使用，其他数据刷新后恢复 mock 状态。
- 不改变页面业务含义，不增加真实接口、遥测或额外网络请求。
- 迁移期间发现的问题仅在与组件迁移直接相关时修复，例如焦点管理、弹层遮挡和图表容器尺寸警告。
- 其他既有缺陷记录在验收结果中，不扩大本次范围。

## 8. 错误处理与无障碍

- 表单提交失败不得静默处理。
- 字段校验错误应靠近字段显示，并通过 Toast 提示提交未成功。
- Dialog、Sheet 和 AlertDialog 使用 Radix 默认焦点锁定、关闭和 Escape 行为。
- 所有纯图标按钮提供明确 `aria-label` 和 Tooltip。
- 键盘用户可以到达并操作菜单、表单、标签页、弹层和危险确认。
- 焦点状态必须清晰可见。
- 本次要求基础可访问性，不进行完整 WCAG 审计。

## 9. 实施顺序

1. 建立 shadcn/ui 基础设施、主题变量、alias 和测试环境。
2. 引入 HashRouter，重构应用入口、顶层路由和 Sidebar。
3. 建立少量业务通用组件。
4. 按业务模块迁移：
   - 工作台
   - 标签中心
   - 人群运营
   - 预警管理
   - 触达运营
   - 数据服务
   - 主题分析
5. 清理已被替代的原生控件样式、自定义弹层和全局导航事件。
6. 集中执行自动化测试、类型检查、构建和浏览器回归。

每个阶段保持改动可检查、可回滚，并在进入下一阶段前通过相关验证。

## 10. 测试方案

测试工具使用 Vitest + React Testing Library。

关键自动化场景：

- 七个顶层路由正常渲染。
- 根路径和未知路径重定向到工作台。
- 侧栏导航、激活状态和折叠行为。
- 标签新增、字段校验、保存和发布。
- 动态人群联动创建触达任务。
- 动态人群联动创建数据服务。
- Dialog、Sheet、AlertDialog 的打开、关闭和焦点行为。
- Toast 的成功、失败和校验提示。
- 已有功能的删除、停用等危险操作确认。

占位按钮只验证能够正常渲染，不要求产生业务结果。

### 11.1 自动化验证

- Vitest 关键流程测试通过。
- TypeScript `tsc --noEmit` 通过。
- Vite 生产构建通过。

### 11.2 浏览器验证

使用最新版 Chrome，在 1280×720 及以上分辨率逐页检查：

- 七个模块均可通过 URL 直达、刷新和浏览器前进后退。
- 页面无新增溢出、遮挡、错位或不可见控件。
- 原有有效交互保持可用。
- 占位按钮保持原有无行为状态。
- 键盘可以操作主要控件，焦点清晰可见。
- Dialog、Sheet、AlertDialog 焦点管理正常。
- 控制台无新增错误和警告。
- 当前 Recharts 容器尺寸警告被修复。

### 11.3 完成定义

满足以下条件才视为完成：

- 全部原生交互控件已按本设计迁移，不长期混用旧组件。
- 七个模块和全局布局均完成迁移。
- 路由替代 `activeMenu + appNavigate`，跨模块联动通过新数据流工作。
- 自动化、类型检查、构建和浏览器回归均通过。
- 未验证项、既有缺陷和剩余风险在最终交付中明确列出。

## 12. 风险与控制

### 12.1 Radix 受控状态迁移

`Select`、`Dialog`、`Sheet` 和 `Tabs` 的受控值接口不同于原生元素。按页面逐项迁移并补交互测试，避免批量机械替换。

### 12.2 路由状态重复消费

`location.state` 若未清除，可能在返回或刷新时重复触发创建动作。目标页面消费后必须 replace 当前历史记录。

### 12.3 紧凑布局回归

shadcn 默认尺寸可能增大页面高度。通过集中小尺寸变体和 1280×720 浏览器验收控制风险。

### 12.4 大页面改动冲突

标签中心、人群运营和主题分析文件较大。迁移时按功能区形成小批次 diff，但不同时进行无关拆分。

### 12.5 占位按钮认知风险

部分按钮仍无实际行为。本次保持兼容，不新增功能；验收记录中明确标记为既有占位能力，避免误判为迁移缺陷。
